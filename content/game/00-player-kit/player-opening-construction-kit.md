# 玩家開局組裝工具

這份不是固定第一章劇本，
而是把開局拆成幾個可替換模組，讓 AI 能依地區與角色組成去組一個像當地、但不死板的開場。

## 預設玩法：白身開局
Elaris 的推薦開局不是：
- 一開場就是成熟職人
- 一開場就有完整身份、執照、穩定聲望

而是：
- 玩家先以 `白身 / 見習 / 跟行 / 候補 / 短工` 的狀態進世界
- 先靠第一批小事、地方麻煩與小委託活進環境
- 再慢慢長成自己真正的職業與生活方式

所以開局時，AI 最先要決定的，不是 `你已經是什麼大人物`，
而是：
- 你現在靠著誰活
- 你正在碰哪條生活線
- 世界第一眼把你當成什麼人

## 開局一定先做的事
1. 選 `種族 / 分支`
2. 用 [起始區域對照表](C:/Users/rabbi/Desktop/Elaris/content/game/00-player-kit/player-start-region-and-opening-pack-matrix.md) 鎖定大區域
3. 確認玩家現在的 `白身狀態`
4. 套該區域的 [地區專屬開局包](C:/Users/rabbi/Desktop/Elaris/content/game/00-player-kit/regional-opening-pack-library.md)
5. 如果該區有既有任務池，就優先從 [開局任務推薦矩陣](C:/Users/rabbi/Desktop/Elaris/content/game/10-quests/player-opening-quest-recommendation-matrix.md) 抽第一批任務

## 白身狀態模組
開局前先替玩家選一種：
- 家傳幫手
- 見習 / 學徒
- 跟行者
- 短工 / 臨工
- 候補 / 未受印

這會決定：
- 第一批 NPC 怎麼看他
- 第一批工作是幫忙、跑腿、照應，還是已可獨立承責
- 玩家是否已經被世界用某個職名呼喚

## 開局模組

### 1. 開場地帶
角色一開始站在哪裡：
- 港邊
- 巢域
- 市集
- 工坊
- 遺跡外圍
- 暫營點
- 邊境哨點
- 前輩住處或工作線邊緣

### 2. 第一個壓力
玩家一開始就會感受到什麼：
- 東西不見
- 路線中斷
- 某人失蹤
- 有人受傷
- 航務卡關
- 物資不足
- 天候或生態異變
- 明明只是小事，卻突然輪到玩家補位

### 3. 第一個互動點
玩家最先接觸誰：
- 港務或城務人員
- 同族長輩
- 公會窗口
- 地方工匠
- 可疑的灰線人物
- 另一組競爭者
- 帶自己入行的前輩或暫時雇主

### 4. 第一個目標
讓玩家知道第一章不是散步：
- 把人找回來
- 把貨送到
- 把路修通
- 把問題查清楚
- 先活過一場短危機
- 先拿到第一份通行或委託
- 先證明你不是來添亂的

### 5. 第一個麻煩
讓開場不會太平：
- 規矩比想像中複雜
- 有人想趁亂撈好處
- 氣候或魔物插手
- 另一個勢力來搶
- 玩家自己的資源不夠
- 玩家其實還沒有完整資格

### 6. 第一個章尾鉤子
讓第一章結尾自然通往下一章：
- 揭出更大的勢力
- 發現失蹤只是表面
- 找到更遠的目的地
- 開啟執照、聲望或長行動線
- 第一次被地方用某個職名叫住

## AI 組裝順序
- 先選 `地帶`
- 再選 `白身狀態`
- 再選 `壓力`
- 再決定 `最先互動的人`
- 接著放入 `第一目標`
- 然後加一個 `麻煩`
- 最後留一個 `章尾鉤子`

## AI 不該做的事
- 不要把開局寫成固定劇本
- 不要玩家一開場就被丟去完全無關的地區
- 不要同時丟太多支線，讓第一章失焦
- 不要跳過地方味，只剩任務框架
- 不要一開場就把玩家寫成高級職人或完整執照持有者

## 白身開局最穩的第一幕組法
- `1` 個起始落點
- `1` 個白身身份
- `1` 個立即壓力
- `2` 個互動點
- `1` 個可接的小任務
- `1` 個後續麻煩
