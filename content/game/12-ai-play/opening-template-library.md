# Opening Template Library

這份文件給 AI / 主持用，目的是讓 `開局` 不會只剩下一句背景介紹。

每個模板都包含：
- 適合的角色組合
- 第一幕應該給的畫面
- 第一個明確目標
- 第一輪可互動物件 / NPC
- 第一個壓力來源
- 第一個失敗也能繼續的鉤子

## 一、官署委託開局

### 適合角色
- 城市型、學術型、調查型角色

### 第一幕畫面
- 一份不完整的文書
- 一位態度保守的司記
- 一個看起來過於普通的委託

### 第一個明確目標
- 找到第一個被隱藏的異常點

### 第一輪可互動
- 司記
- 委託文書
- 廊外等候者

### 第一個壓力來源
- 玩家若動作太慢，相關人物會先一步離場

### 第一個失敗也能繼續的鉤子
- 問不到完整資訊也至少能拿到 `錯誤的名稱 / 殘缺簿頁 / 被塗改的時間`

## 二、潮窟救援開局

### 適合角色
- 海路型、身法型、後勤型角色

### 第一幕畫面
- 回潮倒數
- 失蹤者的殘留裝具
- 港邊人各說各話

### 第一個明確目標
- 在回潮前確認是否入窟

### 第一輪可互動
- 失蹤者同伴
- 港巡兵
- 潮窟口遺留物

### 第一個壓力來源
- 時間壓力直接可見

### 第一個失敗也能繼續的鉤子
- 進窟受阻也能轉成 `外部查線 / 查誰最後見過失蹤者`

## 三、遺跡試探開局

### 適合角色
- 探索、學者、封樣、術式解讀型角色

### 第一幕畫面
- 半開封入口
- 前隊留下的營痕
- 一個過於安靜的風口

### 第一個明確目標
- 判定入口安不安全

### 第一輪可互動
- 井口 / 石門 / 標記
- 遠征隊遺物
- 外圍觀望者

### 第一個壓力來源
- 其他勢力也在等玩家先碰

### 第一個失敗也能繼續的鉤子
- 即使陷阱判錯，也應留下 `誰重新封過這裡` 的線索

## 四、護送路線開局

### 適合角色
- 護衛、巡邏、商旅、引路型角色

### 第一幕畫面
- 看似普通的貨或人
- 安靜得不自然的同行者
- 一條有替代路線的道路

### 第一個明確目標
- 選路並確認護送優先序

### 第一輪可互動
- 雇主
- 車 / 船 / 空艇
- 隨身貨單

### 第一個壓力來源
- 走哪條都不完全安全

### 第一個失敗也能繼續的鉤子
- 選錯路不代表任務終止，而是讓 `敵對者、關卡、環境` 換一種方式出現

## 五、夜拍灰線開局

### 適合角色
- 灰線、商業、估價、情報型角色

### 第一幕畫面
- 燈罩壓低的桌
- 不願報名的人
- 一件真假難辨的貨

### 第一個明確目標
- 判斷交易桌的真正重點

### 第一輪可互動
- 貨盒
- 試價者
- 保桌人

### 第一個壓力來源
- 玩家只要表現得太急，就會先被標記

### 第一個失敗也能繼續的鉤子
- 買不到貨也能換到 `真買家 / 真賣家 / 真來源地` 的線

## 六、極地或荒地求生開局

### 適合角色
- 生存、後勤、遠征、極地型角色

### 第一幕畫面
- 資源不足
- 缺員或故障
- 看似壯麗但實際致命的地貌

### 第一個明確目標
- 先穩住隊伍存活條件

### 第一輪可互動
- 補給堆
- 隊員
- 天候 / 地貌

### 第一個壓力來源
- 環境本身就是敵人

### 第一個失敗也能繼續的鉤子
- 判斷失誤不應直接終局，而應導向 `更差的資源配置 / 更遠的撤退線 / 更糟的天候窗口`

## 七、AI 使用這份模板時的原則

- 開局不要先講百科
- 第一幕要先給 `情境`
- 第一輪一定要有 `可互動物`
- 第一個目標要具體，不要抽象
- 第一個失敗不能直接讓玩家卡死
- 開局三回合內，要讓玩家感覺到自己的種族、職業或背景真的有用
