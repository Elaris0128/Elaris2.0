# 稱號、資格與身分通用框架

## Purpose
這份文件定義 Elaris 的 `稱號 / 資格 / 執照 / 職銜 / 身分位階` 通用規格。

它的目標是讓角色的社會位置能真正改變玩法，而不是只當成好看的稱呼。

## 核心概念
- 稱號不只是名聲，也可能代表責任、許可、限制與可調用資源。
- 同一個角色可以同時持有 `地方稱號`、`職業資格`、`組織職銜` 與 `禮制身分`。
- 有些稱號是榮譽，有些是法定資格，有些則會帶來義務與監管。

## 四大類

### 1. 榮譽稱號
- 來自功績、傳聞、地方認可、重要事件見證。
- 例：邊境守望者、霧路還生者。

### 2. 專業資格
- 來自公會、學會、工坊、軍政系統或特定考核。
- 例：正式藥師、導印技師、認證探索者。

### 3. 組織職銜
- 來自軍團、公會、情報網、王廷與學院。
- 例：分部監記、巡護長、密錄官。

### 4. 制度身分
- 來自主權、盟誓、法律、地方階級或受名體系。
- 例：特許旅人、王路通行持證者、受名見證員。

## 常見取得方式
- 功績授予
- 正式考核
- 組織任命
- 地方見證
- 世襲或血統承襲
- 契約授權
- 特殊事件後的補授

## 常見效果
- 開放某些地區、設施與路線
- 可以收購或使用特定資源
- 降低查驗或談判門檻
- 可接觸更高階任務
- 可請求支援、人手或文件
- 在某些地方會被優待或被敵視

## 常見代價
- 需要服從章程
- 需要接受監督
- 有報告義務
- 有危機時的動員義務
- 容易成為目標
- 若失格會被公告或追責

## 有效範圍
- `地方有效`
- `國內有效`
- `區域互認`
- `世界互認`
- `只在特定組織內有效`

## 狀態標記
- `有效`
- `暫停`
- `受限`
- `撤銷`
- `失格`

## 設計原則
- 稱號最好改變互動，而不只是改變稱呼。
- 資格應與組織、地點、交易、任務和通行真正掛鉤。
- 高階稱號應同時帶來榮耀與壓力。
- 通俗稱呼可以和正式頭銜並存，方便世界更有生活感。

## 稱號卡最低模板
- 名稱
- 類型
- 授予來源
- 有效範圍
- 取得條件
- 主要權益
- 主要義務
- 失格條件
- 外界常見看法

## 與其他框架的關係
- 與 `organization-card-framework.md` 對接：回答哪些組織能發出哪些資格。
- 與 `reputation-framework.md` 對接：回答名聲如何影響頭銜取得與維持。
- 與 `growth-and-advancement-framework.md` 對接：回答成長如何轉成正式身分。
- 與 `value-currency-and-trade-framework.md` 對接：回答頭銜如何影響查驗、稅契與流通權。
- 若要直接引用頭銜與執照示例，應搭配 `title-and-license-example-library.md` 一起使用。
- 若要直接引用玩家常見可持有資格，應搭配 `license-and-qualification-card-library.md` 一起使用。
- 若要快速裁定頭銜帶來的權益、義務與失格後果，應搭配 `title-effect-duty-and-disgrace-matrix.md` 一起使用。
