# 世界組織與勢力索引

## 目的
這個目錄收納 Elaris 中不只屬於單一城鎮的人際與制度性力量，包括國家機關、宗教體系、公會、學會、商會、探索團、情報網與灰色組織。

## 通用入口
- `organization-design-framework.md`：組織設計共通欄位與分類
- `organization-card-framework.md`：組織卡完整版、中型版與輕量版模板
- `non-state-organization-framework.md`：非國家性質組織的共通骨架
- `non-state-organization-archetype-library.md`：非國家性質組織的原型庫
- `non-state-organization-country-relationship-matrix.md`：非國家組織與各國互動矩陣
- `religious-organization-framework.md`：宗教組織作為制度實體時的特殊欄位
- `world-major-religious-orders.md`：世界級主要宗教共同體第一版骨架
- `religions/README.md`：主要宗教的正式神話版本入口
- `intelligence-network-framework.md`：情報網、監察與暗線節點的通用規格
- `merchant-league-framework.md`：商業聯盟、票據網與貨運共同體的通用規格
- `world-major-guilds.md`：世界級主要公會與專業組織
- `non-state/README.md`：具名非國家性質組織的正式入口

## 區域入口
- `east/README.md`：東境組織網與跨國勢力入口

## 使用原則
- 國家文件負責描述「這個國家如何運作」。
- 組織文件負責描述「哪些人透過什麼制度持續影響世界」。
- 同一組織可以同時在多個國家留下分部、特許據點、地下網或合作章程。
- 世界級非國家組織建議至少寫清楚 `總部 / 地方網或分院 / 加入或認證方式`。
- 區域級與地下組織不一定需要正式分部，但建議至少寫清楚 `核心據點 / 影響圈或窗口 / 吸收或進場方式`。
- 若要先做跨國世界感，建議優先定下非國家性質組織，再回頭填國家章程差異。
