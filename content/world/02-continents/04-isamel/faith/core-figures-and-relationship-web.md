# 伊莎梅爾宗教核心人物關係網

伊莎梅爾的宗教衝突很少是單一神職者對單一事件的裁定錯誤，更常見的是 `九域核心人物各自守著正確的一部分`，卻因為價值排序不同而互相拉扯。這份關係網不是官僚名冊，而是主持時可直接拿來判斷 `誰會先聯絡誰、誰會先起疑、誰最容易在公開共議中頂撞誰` 的工具。

## 第一層關係
### 最常互相信任的組合
- `清銘 × 眠誓`
  - 兩者都把 `名字是否被正確送回` 放在極高順位。
  - 一旦出現名錄缺頁、回名錯序、送眠先後爭議，這組最容易形成共同立場。

- `羽祝 × 霧祈`
  - 都傾向先安靜觀察，再決定是否公開升高事件。
  - 對幼龍、鳳裔、照護與靜期秩序的理解最接近。

- `汀祝 × 印沼`
  - 都擅長處理 `資源限取、季節節律、實務使用`。
  - 對濕地、橋泊、祭沼的責任邏輯相容度很高。

- `望翎 × 聽岔`
  - 兩者都重視 `證據與訊號真偽`。
  - 在高風誓望、封路、誤訊與高空危機事件中，常能快速形成判準。

- `炙詠 × 望翎`
  - 都能接受高壓、嚴格、以責任先於舒適的秩序。
  - 在危機期會站得很近，但平時也容易互相競爭正統性。

### 最容易互相拉扯的組合
- `清銘 × 炙詠`
  - 清銘重 `見證完整`，炙詠重 `能否立刻把責任線扶正`。
  - 一個覺得對方太快，一個覺得對方太慢。

- `望翎 × 汀祝`
  - 望翎傾向先封路與先禁行，汀祝傾向保留流動性與地方轉圜。
  - 同一條路在天空與在水路的理解完全不同。

- `羽祝 × 炙詠`
  - 羽祝會先保護照護對象與靜枝秩序，炙詠會先保護整條責任路不崩。
  - 在是否犧牲局部安穩換整體穩定這件事上經常相撞。

- `聽岔 × 清銘`
  - 聽岔要求更多外部可驗證訊號，清銘相信完整見證鏈就足夠。
  - 一旦發生「記錄是完整的，但訊號聽起來不對」的事件，兩人會互相懷疑。

- `霧祈 × 望翎`
  - 霧祈認為不是每個失序都該立刻公開，望翎則極度討厭模糊帶。
  - 在迷霧、封域與高風禁行交疊時特別尖銳。

## 第二層關係
### 重大危機時誰會先找誰
- 清銘通常先找 `眠誓` 或 `汀祝`
- 望翎通常先找 `聽岔` 或 `炙詠`
- 羽祝通常先找 `霧祈` 或 `印沼`
- 汀祝通常先找 `印沼` 或 `聽岔`
- 印沼通常先找 `汀祝` 或 `羽祝`
- 聽岔通常先找 `望翎` 或 `霧祈`
- 炙詠通常先找 `望翎` 或 `清銘`
- 霧祈通常先找 `羽祝` 或 `眠誓`
- 眠誓通常先找 `清銘` 或 `霧祈`

### 誰最常被請來當外域見證
- `眠誓`
  - 因為送眠與回名議題被普遍認為較不易牽扯當下權勢。
- `聽岔`
  - 因為他對訊號與證據真偽很敏感。
- `汀祝`
  - 因為她對地方日用與共同承擔的理解很強，不容易只站高位秩序。

### 誰最容易被懷疑夾帶本域利益
- `清銘`
  - 只要事件牽涉母樹弱鳴，就會被懷疑想替雲瑟拉高解釋權。
- `望翎`
  - 一旦要封風道或高脊，就容易被認為替瑟塔嶺擴權。
- `炙詠`
  - 在演訓、試道、危機期最容易被懷疑借責任之名擴大強制。

## 常見跨域宗教衝突模型
### 名字正確，但責任不正確
- 清銘、眠誓會先說 `名字必須回到正確的位置`
- 汀祝、印沼會追問 `誰實際承擔了代價`
- 炙詠可能會要求先處理秩序崩口，再談完整回補

### 路應該封，還是應該保留
- 望翎會先看高風與高差風險
- 聽岔會先查訊號真假
- 汀祝會先看地方是否還有替代渡行線
- 霧祈會提醒「錯誤資訊」本身也可能是事件主體

### 幼龍、飛龍或鳳裔拒絕新誓
- 羽祝與霧祈通常會先懷疑照護與節點環境出了問題
- 望翎與炙詠則可能先懷疑承誓者資格不足
- 清銘會追問既有見證是否被動了手腳

## 主持用法
- 如果你想做 `安靜但很尖銳` 的宗教戲，先用 `清銘 / 羽祝 / 霧祈 / 眠誓`。
- 如果你想做 `責任、封路、裁定與強硬秩序` 的宗教戲，先用 `望翎 / 聽岔 / 炙詠`。
- 如果你想做 `地方民生、節律、資源與共同承擔` 的宗教戲，先用 `汀祝 / 印沼`。
- 若要讓跨域共議一開始就帶火花，最好把 `清銘、望翎、炙詠` 放在同一場。
- 若要讓玩家感覺自己真的有機會把事情談回來，請安排 `羽祝、霧祈、眠誓` 其中至少一位在場。
