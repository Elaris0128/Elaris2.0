# NPC 角色卡通用框架

## Purpose
這份文件定義 Elaris 的通用 NPC 角色卡寫法，讓國家人物、地方人物、店主、軍官、祭司、情報人與灰線人物都能用同一套欄位建立。

目標不是把每個 NPC 都寫成主角，而是讓不同重要度的角色都能快速對接地點、任務、商店、組織與聲望系統。

## 核心原則
- NPC 必須同時具備 `世界定位` 和 `遊戲功能`。
- 角色卡不要只寫性格，也要寫清楚他能提供什麼、在怕什麼、掌握什麼。
- 大型 NPC 用完整卡，小型 NPC 可用輕量卡，但欄位命名應一致。
- 後續角色若有職業，應同時標記 `通用職業種類` 與 `文化職稱`。

## NPC 分級

### 1. 核心 NPC
適用於：
- 國家要角
- 地區重要引導者
- 主線或長線支線核心人物
- 大城市或大型組織的門面角色

需要完整卡。

### 2. 地方功能 NPC
適用於：
- 店主
- 旅舍主
- 守門官
- 醫療與照護者
- 地方引路人
- 常駐任務派發者

至少要有中型卡。

### 3. 輕量 NPC
適用於：
- 市場攤販
- 村中代表人物
- 短線支線人物
- 一次性見證人或消息來源

可以用簡卡，但應保留功能欄。

## NPC 功能分類
後續建立角色時，建議至少勾一到兩種功能位：
- `任務派發`
- `情報供應`
- `商店 / 服務`
- `通行 / 查驗`
- `醫療 / 安養`
- `工藝 / 製作`
- `組織代表`
- `地方人脈`
- `戰鬥支援`
- `灰線交易`
- `文化說明`

## 完整角色卡建議欄位

### Metadata
- id
- region
- nation
- location
- faction
- role
- status

### Snapshot
- 年齡
- 種族
- 種族-分支
- 通用職業種類
- 文化職稱
- 等級 / 危險度
- 關鍵能力

### Public Impression
- 外觀或第一印象
- 他人通常怎麼看他
- 初見時最明顯的特徵

### Personality
- 性格傾向
- 價值觀
- 明顯好惡

### Speech Style
- 語速
- 用詞傾向
- 口頭禪或常見表達

### Background
- 出身
- 經歷
- 現在的立場與位置

### Hidden Layer
- 秘密
- 真正目的
- 不願被碰到的弱點

### Resources and Reach
- 他掌握的東西：人脈 / 物資 / 地點 / 文件 / 權限 / 魔物 / 交通
- 他能影響哪些系統：商店 / 任務 / 通行 / 聲望 / 情報

### Relationships
- 公開盟友
- 私下合作對象
- 明面敵對
- 欠人情 / 被欠人情

### Game Hooks
- 可提供的任務
- 可出售或可交換的特殊內容
- 與玩家互動後可能解鎖的後續

## 中型角色卡最低欄位
- 名稱
- 所在地
- 身分 / 角色
- 通用職業種類
- 關鍵能力
- 性格一句話
- 能提供什麼
- 在隱瞞什麼
- 任務或交易鉤子

## 輕量角色卡最低欄位
- 名稱
- 所在地
- 功能位
- 一句話印象
- 可互動內容

## 商店 NPC 額外欄位
若角色是店主、掌櫃、攤主或服務提供者，建議另外補：
- 招牌貨或招牌服務
- 收購方向
- 對熟客與陌生客的差異
- 缺貨或限量條件
- 最容易聊到的地方消息

## 組織 NPC 額外欄位
若角色屬於國家、軍方、公會、教團或情報體系，建議另外補：
- 所屬組織
- 正式權限
- 私下權限
- 需要遵守的規程
- 最怕哪條規程被查

## 關係寫法建議
關係最好不要只寫 `朋友 / 敵人`，而是寫成：
- 為什麼合作
- 為什麼敵對
- 誰欠誰什麼
- 一旦局勢變動，誰最可能先翻臉

## 與地方包對接
- 核心地點建議把重要 NPC 拆成獨立角色卡
- 小型村聚可先把 NPC 摘要放在地方包內，之後再視重要性拆卡
- `locations/*.people.md` 應優先引用這套欄位語言

## 與職業系統對接
- 角色若有專業身分，應優先標記 `通用職業種類`
- 地方稱呼、官銜或公會稱號則放在 `文化職稱`
- 同一 NPC 可以同時有主職與副職，但不要把所有能力都塞滿

## 設計原則總結
- 角色卡應讓主持人一眼知道：這人是誰、能幹嘛、為什麼重要。
- 對文字冒險來說，`可互動性` 和 `可延展性` 比傳記長度更重要。
- 真正好用的 NPC，通常是同時能提供一項幫助，並帶出一項麻煩的人。

## 延伸規格
- 若要進一步處理人物之間的債務、派系站位與關係翻轉，應搭配 `npc-relationship-framework.md` 一起使用。
- 若要快速套用常見角色功能位與店型，應搭配 `npc-function-and-shop-archetype-library.md` 一起使用。
