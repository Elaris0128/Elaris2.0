# 世界整體架構

## Purpose
這份文件整理 Elaris 的 `整體世界架構`，給作者、主持與世界觀整理時使用。

它回答的是：
- 這個世界的底層結構要怎麼被分層理解
- 哪些內容屬於世界真實層
- 哪些內容屬於文明詮釋層
- 各模組在世界中的位置是什麼

這份文件不是給玩家直接閱讀的導覽文，而是整個專案的 `世界總藍圖`。

## 核心原則
- 世界真實層與玩家認知層要分開。
- 各國、各宗教、各種族可以有不同說法，但不能互相打碎通用底層。
- 世界架構先固定，玩家之後再透過遊戲逐步接觸其中一部分。
- 各地區與國家內容，應視為這份總架構的局部實作，而不是平行宇宙。

## 一、世界分層

### 1. 世界真實層
- 包含創世主、七大高位神、權柄層級、死後世界共同底層、靈息與術式結構、種族與文明分布、生態與資源骨架。
- 這一層不因地區不同而變成不同宇宙。
- 主檔入口：
  - `god-system-framework.md`
  - `creator-and-high-gods.md`
  - `authority-hierarchy-and-branching-framework.md`
  - `afterlife-and-soul-cycle-framework.md`
  - `magic-system-index.md`

### 2. 文明詮釋層
- 包含宗教神話、種族神話、地方祭儀、國教、祖靈傳承、氏族故事與地方世界觀。
- 這一層可以彼此不同，甚至互相衝突，但它們通常是在詮釋同一份世界真實層。
- 主檔入口：
  - `religion-and-faith-framework.md`
  - `../04-factions/religions/README.md`
  - `../05-races/myths/README.md`

### 3. 政治與組織層
- 包含國家、聯盟、山邦、棲國、世界級公會、商會、情報網與地下組織。
- 這一層決定世界如何被管理、流通、查驗、護送、滲透與利用。
- 主檔入口：
  - `polity-and-civilization-framework.md`
  - `../04-factions/README.md`

### 4. 生態與棲地層
- 包含魔物生態、資源鏈、特殊地點、聚落與旅行節奏。
- 這一層決定玩家走到哪裡會遇到什麼、能採到什麼、要怕什麼。
- 主檔入口：
  - `monster-ecology-framework.md`
  - `global-resource-taxonomy.md`
  - `special-site-framework.md`
  - `settlement-design-framework.md`
  - `travel-survival-and-exploration-framework.md`

### 5. 遊戲運作層
- 包含職業、能力、特質、負荷、裝備、交易、任務、成長、權柄承載與 AI 主持規格。
- 這一層決定玩家怎麼玩，而不是世界在本體上怎麼存在。
- 對應入口：
  - `../../game/01-character/job-system.md`
  - `../../game/02-stats/ability-framework.md`
  - `../../game/07-growth/growth-and-advancement-framework.md`
  - `../../game/12-ai-play/README.md`

## 二、世界的五大主軸

### 1. 空間主軸
- 世界由大陸、群島、洋域、霧林、山域、深廊、河網、邊境古道與特殊異常帶組成。
- 不同國家與文明，是在同一張空間結構上佔據不同位置。
- 除了一般大陸與國家，也存在 `曦環浮境 / 奧崙群星帶 / 六大洋 / 亡極徹淵 / 群島弧帶` 這類需要以獨立世界區域來理解的高權重舞台。

### 2. 文明主軸
- 文明單位包括國家、聯邦、聯盟、山邦、棲國、氏族地與地方共同體。
- 種族不等於國籍，信仰不等於政治體，文化不等於血統。

### 3. 神聖主軸
- 創世主與高位權柄是世界共同底層。
- 各宗教、各種族與地方信仰，是對同一套神聖結構的不同詮釋。
- 神不降臨，凡人不直見神。

### 4. 生態主軸
- 世界的自然生命以 `魔物生態` 統攝。
- 魔植、魔菌、魔蟲、獸型、類人、棲地王者、伴生與家用魔物，共同構成文明的日常環境。

### 5. 可玩主軸
- 玩家不是全知觀察者，而是從某個地方、文化、職業與信仰背景進入世界。
- 因此遊戲呈現應該由局部視角逐步打開，而不是一次攤開全世界真相。

## 三、從上到下的內容關係
- 世界整體架構
- 世界通用框架
- 大區域
- 特殊區域 / 洋域 / 浮境 / 深淵域
- 國家 / 文明單位
- 地方包 / 聚落 / 特殊地點
- 組織 / NPC / 商店 / 任務 /遭遇
- 玩家當下認知與互動

這個順序的作用是：
- 上層決定下層能長成什麼樣
- 下層補充上層在地化細節
- 不讓地方設定反過來破壞世界底層邏輯

## 四、哪些內容要先固定，哪些可以延後揭露

### 先固定
- 世界真實層骨架
- 空間與文明層級
- 神系與死後世界共同底層
- 靈息、術式、戰技、成長與交易的共通規格
- 組織、公會、國家與灰線網的基礎存在方式

### 可以延後揭露
- 各國如何講自己的創世史
- 地方神話與異端版本
- 某些下位神、地方聖地與祖靈的真正來歷
- 黑市與地下組織的深層結構
- 特定王者級魔物與特殊地點的真相

## 五、作者 / 主持 / 玩家三種視角

### 作者視角
- 可以看見完整世界架構。
- 用來保證設定一致、避免返工。

### 主持視角
- 需要知道世界底層與玩家目前可知層的差距。
- 不能把作者知道的全部直接倒給玩家。

### 玩家視角
- 只能從出身、地區、職業、信仰、組織、探索與事件逐步得知世界。
- 看到的是「世界的一部分」，而不是世界全貌。

## 使用原則
- 每新增一個區域、國家或大型組織時，都應先確認它在這份架構中的位置。
- 若某個新設定只能在地方成立、但會推翻整體底層，應視為需要重新討論的高風險變更。
- 若某個新設定只是地方神話、地方偏見或地方知識限制，則應留在玩家認知層處理，不要直接動世界真實層。
